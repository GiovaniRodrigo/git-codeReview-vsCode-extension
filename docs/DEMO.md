# Demo

## Local Git Review

1. Open a Git repository in VS Code.
2. Open the Source Control container.
3. Use the `Git Review` view to expand local branches, remote branches, or tags.
4. Expand a commit to see changed files.
5. Select `Open File Diff` for the native VS Code diff.
6. Select `Open GitHub Style Diff` for the patch-style review panel.
7. Select `Open Review` on a branch, tag, or commit to use filters, search, statistics, and multi-select.

## GitHub Pull Requests

1. Use `Code Review: Sign In to GitHub`.
2. Refresh the `Git Review` view.
3. Expand `Pull Requests` to see PRs, state, and checks.
4. Use PR actions to comment, approve, or request changes.

## Productivity

- Use `Code Review: Compare Branches` to inspect branch diffs.
- Use `Mark Commit Reviewed` and `Show Unreviewed Commits` to keep local review progress.
- Background refresh detects new commits in the active repository.
